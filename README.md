# 🏦 FinSecure Bank - Access Control Strategy

## 📋 Project Description

This project presents a **hybrid access control strategy (RBAC + ABAC)** for a mid-sized financial institution. The interactive website showcases:

- Organization with 800 employees and 5 departments
- Comparison of access control strategies
- Contextual access decision examples
- Implementation challenges and solutions
- Data sensitivity classification
- Implementation roadmap
- Regulatory compliance (GDPR, PCI-DSS, SOX, GLBA)

## 🚀 Technologies Used

- HTML5
- CSS3 (with futuristic, neon, and glitch effects)
- JavaScript (animations, particles, interactivity)
- 100% Responsive Design

## ✨ Key Features

- **Futuristic design** with neon and glitch effects
- **Fully responsive** (mobile, tablet, desktop)
- **Bilingual** (Spanish/English with language selector)
- **Hamburger menu** for mobile devices
- **Scroll animations**
- **Dynamic background particles**
- **Animated counters** for statistics

## 🎯 Strategy: Hybrid RBAC + ABAC

| RBAC | ABAC |
|------|------|
| Role-based permissions | Context-aware decisions |
| Easy to manage | Granular control |
| Organizational mapping | Dynamic risk adjustment |

## 📊 Sensitivity Levels

- 🔴 **CRITICAL**: Mergers, board meetings, CEO credentials
- 🟠 **HIGH**: Financial data, employee PII
- 🟡 **MEDIUM**: Internal communications, reports
- 🟢 **LOW**: Public materials

## ✅ Regulatory Compliance

- **GDPR** (Article 25): Privacy by design
- **PCI-DSS** (Requirement 7): Restrict cardholder data
- **SOX** (Section 404): Internal controls
- **GLBA** (Safeguards Rule): Customer information protection
